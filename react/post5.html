# 리액트 LifeCycle

# Overview

  • 컴퍼넌트가 생성되고 소멸되는 과정을 LifeCycle이라고 한다.
  • React는 LifeCycle의 특정시점에 실행되는 API(매서드)를 제공하고 있다.

# 기본적인 흐름 이해하기

  • 그림에서 가장 많이 차지하고 있는 render 이 컴퍼넌트가 랜더링되는 시점이다.
  • 그래서 render시점 이전인 consructor 에서 status값을 초기화 한다.
  • 또한 consructor 시점에서 'super(poops)'로 부모의 props를 상속받거나 'this'를 bindding하게된다.
  • render 가 되기전에는 status값을 변경해서는 안된다.
  • render이 완료된 시점인 componentDidMount, componentDidUpdate 시점에서 status를 변경할 수 있다.
  • AJAX요청 및 상태의 변경은 보통 componentDidMount에서 처리하게 된다.
  • componentWillUnmount 는 컴퍼넌트가 소멸되고 실행된다.

# 가끔식 사용되는 LifeCycle methods

  • componentWillReceiveProps : 컴퍼넌트가 porps를 새로 전달받았을때 실행되며 리랜더링 되지 않는다.
  • shouldComponentUpdate는 Boolean값을 반환하는데 porps나 status가 변경되었을때 랜더링의 유무를 결정한다.

# React16 부터 추가된 LifeCycle methods

  • getDerivedStateFromProps : 새로운 porps전달 받았을 때 실행되며 componentWillReceiveProps의 대안으로 추가되었다.
  • getSnapshotBeforeUpdate : render 이후 DOM이 변화하기 직전에 실행되며 기존의 componentWillUpdate의 대안으로 추가되었다.

# 결론

  • LifeCycle은 학습하면서 문서나 책을 통해 정확히 이해할 수 없었고 기본적인 LifeCycle Methods를 제외하고 특빌히 적용될만한 상황을 찾기가 어려운 것 같다.
  • 하지만 큰 흐름을 알고있다면 필요한 순간에 찾아서 적용하면 될 것 같다는 느낌이 든다.
