# 리액트상태관리 with MobX

# Overview

  • 리액트는 컴퍼넌트의 변화된 상태를 porps로 전달하는데 컴퍼넌트 구조가 복잡해지면 상태관리에 대해서 다른 접근이 필요하다.
  • Redux와 MboX가 많이 사용되는 상태관리인데 여기서는 좀 더 간단하고 최신기술인 mbox의 가장 기본적인 사용법만 정리하였다.

# install

  • create-react-app으로 프로젝트를 생성 후 아래의 모듈을 추가로 설치한다.
npm install mobx mobx-react
  • 아래와 같이 eject하여 preset 설정을 추가한다 (데코레이터를 사용하기 위함)
npm install babel-preset-mobx
npm run eject

  //package.json
 "babel": {
    "presets": [
      "react-app",
      "mobx"
    ]
  }

# concept basic

  • store에서 observable데이터를 만들고 관리한다. (@obserable)
  • component는 store의 observable 데이터를 구독한다. (@inject / @observer)
  • 데이터의 변경은 store에서 일괄 관리한다. (@action)
  • store는 데이터를 연산하여 전달할 수 있다. (@computed)

# 참고

  • @action에서 비동기 데이터를 처리할 수 있다.
  • autorun기능을 활용하면 다양한 기능을 구현할 수 있다.
  • @obserable은 Proxy객체로 변환되어지는데 option값으로 이를 변경할 수 있다.
  • 기본을 바탕으로 다양한 기능을 문서를 통해 활용하며 배워가거나 이미 작성된 코드분석을 통한 지속적인 학습이 필요할 것 같다.
