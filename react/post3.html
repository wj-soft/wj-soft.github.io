# 리액트 라우터

# Overview

  • 프론트앤드 또는 리액트에서 라우터는 요청하는 url에 값에 따라 다른 컴퍼넌트로 view를 전환하는 것을 의미한다.
  • react-route를 사용하면 SPA에서 다양한 페이지처럼 보여줄 수 있다.
  • react-route는 리액트에 공식적으로 포함된 기능은 아니고 서드파티라이브러리로이다.
  • 이글은 react-route의 최신버전인 v4의 가장 기본적인 내용만 담아서 작성되었다.

npm install react-router-dom

# react-router basic

  • 요청한 url에 따라 컴퍼넌트가 보여지는 영역을 설정한다.
import React, { Component } from 'react';
import { BrowserRouter as Router, Route, Switch} from 'react-router-dom';

import Home from './pages/Home';
import About from './pages/About';

class App extends Component {
  render() {
    return (
      <Router>
        <div>
          <Switch>
            <Route exact path="/" component={Home}></Route>
            <Route path="/about" component={About}></Route>
            <Route render={() => <div>일치하는 페이지가 없음</div>} />
          </Switch>
        </div>
      </Router>
    );
  }
}

export default App;
  • url path에 따라 Home 또는 About컴퍼넌트를 전환하여 보여준다.
import React, { Component } from 'react';
import { Link } from 'react-router-dom';

class About extends Component {
  render() {
    return (
      <div>
        About페이지
        <h1><Link to="/">Home으로 가기</Link></h1>
      </div>
    );
  }
}

export default About;
  • 페이지의 이동은 위와 같이 작성한다.

# params와 query

  • url에 동적으로 params를 부여할 수 있다.
<Route path="/item/:id" component={Item} />
  • params는 props를 통해 아래와 같이 컴퍼넌트에서 값을 취득할 수 있다.
import React, { Component } from 'react';

class Item extends Component {
  render() {
    return (
      <div>
        Items : {this.props.match.params.id}
      </div>
    );
  }
}

export default Item;
  • query의 경우 this.props.location.search에 문자열로 값을 가지고 있다.
  • query의 값이 필요한 경우 추출하는 로직을 새로 만들거나 query-string 라이브러리를 사용한다.
  • 라이브러리 : https://github.com/sindresorhus/query-string
