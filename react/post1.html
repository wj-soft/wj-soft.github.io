# 리액트 시작하기

# Overview

  • 리액트는 view를 관리하기 위한 Javascript라이브러리
  • Virtual DOM 기술을 사용함
  • JSX로 View를 표현함 (vue는 HTML 템플릿 기반)
  • ES6를 사용하기 때문에 트랜스파일링(babel)이 필요함
<head>
  <meta charset="utf-8">
  <meta name="viewport" content="width=device-width">
  <title>Studying</title>
  <script src="https://unpkg.com/react@15.3.2/dist/react.js"></script>
  <script src="https://unpkg.com/react-dom@15.3.2/dist/react-dom.js"></script>
  <script src="https://cdnjs.cloudflare.com/ajax/libs/babel-core/5.8.23/browser.min.js"></script>
</head>
<body>
  <div id="root"></div>
  <script type="text/babel">
     리액트 코드 작성....
  </script>
</body>
  • 일반적으로 Node.js환경에서 컴포넌트 단위로 모듈화하며 create-react-app으로 빠르게 환경을 구성할 수 있음
//패키지 설치
npm install -g create-react-app
//프로젝트 만들기
create-react-app [프로젝트이름]
//프로젝트폴더로 이동
cd [프로젝트이름]
// http://localhost:3000/ 에서 실행
npm start

# 컴퍼넌트 (component)

  • 정의 : 화면을 구성하는 작은 단위
  • 정리하면 다양한 컴퍼넌트들의 조합으로 view를 만들고 이 과정에서 컴퍼넌트는 값 또는 상태를 공유한다
  • Component는 Javascript로 만들어지는데 이때 view를 만들 때 HTML과 유사한 구조로 만들어진 JSX를 활용한다.
class Hello extends React.Component {
  render() {
    return <div>Hello {this.props.toWhat}</div>;

    //without JSX
    // return React.createElement('div', null, `Hello ${this.props.toWhat}`);
  }
}

ReactDOM.render(
  <Hello toWhat="World" />,
  document.getElementById('root')

  //without JSX
  // React.createElement(Hello, {toWhat: 'World'}, null),
  // document.getElementById('root')
);
  • 일반적인 경우 컴퍼넌트는 class로 정의하지만 상태관리나 라이프사이클이 필요없는 간단한 컴퍼넌트는 함수로 정의한다
function Hello() {
  return (
    <div>Hello {this.props.toWhat}</div>
  );
}

# JSX

  • 컴퍼넌트를 만들 때 사용된 return 값이 JSX이고 HTML과 형태가 유사하지만 Javascript의 확장된 문법이다 공식문서:링크

  • 어려운 내용은 없으니 위에 링크된 공식문서를 읽어보면 쉽게 이해할 수 있고, HTML과 다른 몇가지 특징을 나열하면 다음과 같다

    • 중괄호 '{}' 에는 자바스크립트 값이 들어온다.
    • class는 className로 적는다.
    • 모든 태그는 닫혀있어야 한다.
