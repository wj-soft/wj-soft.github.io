# 리액트 porps와 state, event

# Overview

  • props는 부모컴퍼넌트가 자식컴퍼넌트에게 전달하는 값을 의미한다.
  • state는 동적인 데이터들의 객체로 this.setState로 값을 변경할 수 있다.
  • react에서 데이터는 단방향(위에서 아래로)으로 흐른다.
  • 정리하면 컴퍼넌트의 state가 변경되고 변경된 state는 porps로 하위컴퍼넌트로 전달한다.
  • 일반적으로 어플리케이션은 event에 의해 상태값에 변화가 일어나고 props로 다른컴퍼넌트에 변경내용을 전달하는 구조를 가진다.

# porps

class User extends React.Component {
  render() {
    return <div>Welcome {this.props.name} {this.props.sayHi()}</div>;
  }
}

ReactDOM.render(
  <User name="lee"
  sayHi={()=>'hi'} />,
  document.getElementById('root')
);
  • 위 코드에서 User컴퍼넌트는 상위컴퍼넌트로부터 데이터를 name과 sayHi라는 값을 props로 받고있다.
  • 함수도 1급객체로서 props로 전달 할 수 있다.
  • porps를 간단하게 표현하면 상위컴퍼넌트로부터 전달되는 모든 값을 의미한다.
  • react에서는 props의 타입을 체크하고 defaultProps을 정의할 수 있다. (v15.5 이상부터 라이브러리 추가 없이 사용가능)
import React, { Component } from 'react';
import PropTypes from 'prop-types';

class Hello extends Component {
  static defaultProps = {
    xyz: 'String'
  };

  static propTypes = {
    abc: PropTypes.number,
    xyz: PropTypes.number
  };
  
  render() {
    return (
      <span>{this.props.abc} {this.props.xyz}</span>
    );
  };
}

export default Hello;
  • 위 코드의 경우 xyz를 defaultProps로 받게 되는데 type이 일치하지 않기 때문에 console창에 에러가 발생하게 된다.

# state

  • state는 컴포넌트가 가지고 있는 변경되는 데이터이다.
  • this.setState로 상태값을 변경한다.
import React, { Component } from 'react';

class App extends Component {
  state = {
    abc: 'hello',
    def: 'world'
  }

  stateChange = () => {
    this.setState({
      abc: 'hi'
    })
  }

  render() {
    return (
      <div className="App">
        {this.state.abc}  {this.state.def}
        <button onClick={this.stateChange}>상태값 바꿔보기</button>
      </div>
    );
  }
}

export default App;
  • 위 코드는 버튼을 클릭하면 상태값 변경을 확인해보는 간단한 코드이다.
  • 전달받은 props를 가지고 상태값을 경우 아래와 같이 수정하여 사용한다.
stateChange = () => {
    this.setState((state, props) => ({
      abc: 'hi' + props.propsName
    }));
  }
  • props를 함수의 파라미터로 전달하였는데 this.props.propsName을 사용해서는 안된다.
  • 마지막으로 아래와 같이 다른 로직과 merged 될 수 있다.
  //react documents (https://reactjs.org/docs/state-and-lifecycle.html)
  componentDidMount() {
    fetchPosts().then(response => {
      this.setState({
        posts: response.posts
      });
    });

    fetchComments().then(response => {
      this.setState({
        comments: response.comments
      });
    });
  }

# 이벤트 (공식문서 참조)

링크:공식문서

  • React는 이벤트명은 카멜케이스로 작성하고 함수명을 전달한다.
<button onClick={activateLasers}>
  Activate Lasers
</button>
  • 다른 기본동작을 막기 위해 preventDefault을 사용한다.
function ActionLink() {
  function handleClick(e) {
    e.preventDefault();
    console.log('The link was clicked.');
  }

  return (
    <a href="#" onClick={handleClick}>
      Click me
    </a>
  );
}
  • 중첩된 요소에서 이벤트의 전파를 막기 위해서는 stopPropagation를 사용한다.
  <button onClick={e => {
      this.props.removeHandler(v.pk);
      e.stopPropagation();
    }}>Click</button>
  • this의 의미는 cnstructor에서 bind하거나 화살표함수로 매서드를 정의하여야 한다.
class Toggle extends React.Component {
  constructor(props) {
    super(props);
    this.state = {isToggleOn: true};

    // This binding is necessary to make `this` work in the callback
    this.handleClick = this.handleClick.bind(this);
  }

  handleClick() {
    this.setState(prevState => ({
      isToggleOn: !prevState.isToggleOn
    }));
  }

  render() {
    return (
      <button onClick={this.handleClick}>
        {this.state.isToggleOn ? 'ON' : 'OFF'}
      </button>
    );
  }
}

ReactDOM.render(
  <Toggle />,
  document.getElementById('root')
);
  • 합수에 파라미터를 전달하가 위해서는 아래의 형태를 사용해야 한다.
<button onClick={(e) => this.functionName(value, event)}>Delete Row</button>
